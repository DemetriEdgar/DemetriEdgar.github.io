AVANFOR/아픈 반려견을 위한 일정관리


시연영상


주제선정

현재 세 가구 중 한 가구는 반려견을 키우는 시대입니다. 따라서 반려견을 위해 백신 접종 주기나 먹이를 주거나 아플 때 일정을 잊어버리지 않고 관리해 줄 수 있는 웹/모바일 앱이 있으면 좋겠다는 생각에 주제를 선정하게 되었습니다.


프로젝트 기능 및 설계

- 언어 및 툴

Spring MVC / Java / Oracle / Apache Tomcat / jsp / HTML / CSS / js / Android Studio / Maven / Mybatis / Firebase / Bootstrap / Kakao API / Ajax / jQuery / JSON / Font Awesome

- 주요 기능

일정관리 / 모바일 앱 및 앱 푸시알림 / 게시판 / 회원가입, 로그인 / 주변병원 찾기

- 설계 방식

Spring 프레임워크를 이용, STS툴로 MVC 패턴을 사용하여 구현하였습니다. DB는 Oracle을 사용하였고 서버는 Apache Tomcat을 사용하였습니다. 프론트엔드는 HTML, CSS, JSP, Bootstrap, Font Awesome, jQuery을 이용하여 구현하였으며 주변병원 찾기의 지도는 카카오맵 API를 이용하였습니다. 일정관리의 캘린더는 Full Calendar를 이용하였으며 모바일 앱은 안드로이드 스튜디오를 통해 웹뷰를 사용하여 하이브리드 앱으로 구현 하였습니다. 일정에 등록한 알람을 위하여 컨트롤러에서 해당 조건 충족 시 웹서버를 통해 Firebase에 FCM 전송을 요청하여 모바일 앱에 푸시 알림이 가도록 구현하였습니다.


프로젝트 일정 / ERD / 코드 컨벤션 및 네이밍 룰

역할 및 진행소감

팀프로젝트는 총 3명으로 진행하였으며 제 역할은 PM과 통합테스트 담당을 맡았습니다. 개발 파트는 프론트 / 백 / DB 등으로 나누지 않고 각자가 구현하고자 하는 기능을 맡아 각 기능에 해당하는 프론트 / 백 / DB를 모두 구현하였습니다.
따라서 프로젝트 일정에도 프론트/백/App의 개발기간이 동일합니다. 제가 맡은 기능은 모바일 앱 제작, 게시판 및 댓글, 모바일 푸시알림 기능, 푸시알림을 위한 캘린더의 데이터를 이용한 스케줄링, 스프링 웹서버-Firebase-안드로이드 앱 연동 및 각 기능에 해당하는 DB 구축을 맡았습니다.
통합테스트는 정기적으로 주 1회 진행하였고 각 기능이 완성되면 수시로 통합테스트를 진행하였으며 메신저 또는 메일을 통해 협업을 하였습니다.
그 동안 여러 팀프로젝트를 하면서 각자가 각 기능의 모든것을 도맡아 진행한 것은 처음이였지만 전체적으로 복습하는 상황이 되어 좋은 도움이 되었습니다. 그동안 거의 대부분의 프로젝트에서 팀장 또는 PL의 역할을 맡았었는데 이번 팀프로젝트는 코로나 시국이라 대부분 온라인으로 협업을 하였기에 온라인으로 통합테스트 및 형상관리를 한 것이 색다른 경험이었으며 PM으로서 해야할 역할에 대해 많이 배웠습니다.
